# Actuarial Model Parity Contract

**Version:** 1.0  
**Last updated:** 2026-03  
**Applies to:** SPIA ALM projection engine (Python ↔ Excel)

This document is the authoritative source of truth for numerical parity between the Python
calculation engine (`pricing_projection.py`) and the Excel workbook generated by
`alm_excel_ladder.py` / `build_pricing_excel_workbook.py`.

Any change to calculation logic must be reflected in **both** engines and must be tested
using the suite in `tests/parity/`.

---

## 1. Parity Definition

For any given set of inputs, the Python engine and the recalculated Excel workbook must
agree to within the tolerances below at **every month** of the projection horizon, for
every tracked state variable.

| Variable | Absolute tolerance | Notes |
|----------|--------------------|-------|
| Cash balance EOM | 1e-4 | Dollar |
| Bond face EOM | 1e-4 | Dollar per bucket |
| Bond MV EOM | 1e-4 | Dollar per bucket |
| Total asset MV | 1e-4 | Dollar |
| Liability PV | 1e-4 | Dollar |
| Surplus | 1e-4 | Dollar |
| Remaining tenor (t_rem) | 1e-6 | Years |
| Discount factor (DF) | 1e-10 | Dimensionless |
| Reinvest amount (dmv) | 1e-4 | Dollar |

> **Display rounding** (2 decimal places for $ values in the UI) is the only place where
> values may be rounded. All internal calculations use full float64 precision.

## 2. Tie-Break Policy

When two or more bond buckets have equal or near-equal remaining tenor, the **lower-indexed
bucket is always liquidated first**. This applies to both disinvestment and any future
rebalancing logic.

### Python implementation
```python
order = np.argsort(
    t_rem
    + (faces <= 1e-15) * 1e6          # push depleted buckets to end
    + np.arange(faces.shape[0]) * 1e-9  # epsilon tie-break by index
)
```

### Excel implementation
```
trnk[k] = t_re[k] + (k+1)*1E-9
tmin     = MIN(IF(face[k]>1E-9, trnk[k], 999) for each k)
sell_k   = IF(ABS(trnk[k] - tmin) < 5E-10, ...)  ← threshold = half the epsilon interval
```

**Critical**: The comparison threshold in Excel must be **strictly less than half the
minimum epsilon interval** between consecutive buckets (`5E-10 < 1E-9 / 2`). If the
threshold equals or exceeds the epsilon interval, two adjacent buckets can be selected
simultaneously when their `t_rem` values differ by floating-point noise (~1e-16).

## 3. Epsilon Policy

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Per-bucket epsilon (Python) | `k * 1e-9` | Ensures unique sort key per bucket |
| Per-bucket epsilon (Excel) | `(k+1) * 1e-9` | k=0 gets non-zero epsilon for tmin comparison |
| Comparison threshold (Excel) | `5e-10` | Half the inter-bucket interval |
| Depleted-bucket sentinel (Python) | `1e6` | Pushes zero-face buckets to end of order |
| Depleted-bucket sentinel (Excel) | `999` | Pushes zero-face buckets to end via IF guard |

**Do not change epsilon values** without:
1. Updating this table.
2. Running all golden parity scenarios.
3. Adding a boundary test proving no double-trigger for the new values.

---

## 4. Floating-Point Accumulation Warning

Bond `t_rem` values computed by repeated subtraction of `dt = 1/12` from an initial value
WILL differ from `t_rem` values that were reset to the nominal tenor after reinvestment,
even when both should represent the same time-to-maturity.

**Example (the 2026-03 bug):**  
- 1Y bond reset to `1.0` at month 24, decremented 11 more times → `0.08333333333333325`  
- 3Y bond decremented 35 times from `3.0` → `0.08333333333333247`  
- Raw difference: `~7.8e-16` — below any economic tolerance but **flips argsort order**

**Rule**: Never use raw `t_rem` comparison for ordering. Always use epsilon-adjusted keys.

## 5. Yield Curve / Discount Factor Policy

- Interpolation method: **log-linear** on zero rates (continuously compounded).
- Boundary handling: flat extrapolation beyond the longest available maturity.
- Both engines use the same formula: `DF(t) = exp(-(z_lo + w*(z_hi - z_lo)) * t)` where
  `w = (t - t_lo) / (t_hi - t_lo)`.
- Spread is applied as an additive shift to the zero rate before computing DF.

---

## 6. Reinvestment Policy

- Rule: `pro_rata` — excess cash reinvested proportional to gap-to-target weight.
- Triggered **only** when at least one bond matures in the current month.
- Post-maturity `t_rem` reset: set to nominal tenor for all buckets where `f_pm <= 1e-9`
  AND `t_pm <= 1e-9` (handles both matured-and-depleted and already-depleted buckets).

---

## 7. Known Risk Areas (lessons learned)

| Area | Failure mode | Mitigation |
|------|-------------|------------|
| Disinvestment tie-break | Double-sell when two bonds have t_rem differing by ~1e-16 | 5e-10 threshold; epsilon argsort |
| dmv formula in Excel | Divide-by-DF bug converts MV amount to face amount | Unit test: `mv = face * df`, not face |
| t_rem reset gating | Wrong reset condition misses depleted bonds, causing t_rem accumulation error | Gate on `f_pm <= 1e-9 AND t_pm <= 1e-9` |
| OOXML cache injection | Stale cached values shadow formula results in data_only reads | Always open Excel for full recalculation; never rely on data_only for formula cells |
